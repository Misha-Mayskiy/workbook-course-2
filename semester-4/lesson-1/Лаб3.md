# Лабораторная работа: «Абстрактные классы и Полиморфизм. Создаем ASCII-Подземелье»

### Цель работы:
В предыдущей работе вы узнали, как вручную управлять памятью. Теперь мы применим эти знания для создания архитектуры игры. 
Представьте, что у вас есть карта 10х10. На ней могут быть Стены, Пол, Золото, Ловушки. Если писать код «в лоб», вам придется делать кучу проверок: `if (cell == "Wall") ... else if (cell == "Gold") ...`. Это плохой код.

В C++ мы можем создать массив **указателей на базовый класс**, положить туда разные объекты-наследники и дергать их за одинаковые «ниточки» (виртуальные методы). Объекты сами будут решать, как им выглядеть и что делать!

---

### Шаг 1: Игрок и Базовый класс (Абстракция)

Для начала создадим простую структуру Игрока. Напишите её над всеми остальными классами:
```cpp
struct Player {
    int x = 1;
    int y = 1;
    int hp = 100;
    int gold = 0;
};
```

Теперь создадим интерфейс для любой клетки на карте. 
Создайте класс `Cell`. В нем не будет никаких данных (полей), только правила для наследников:

1. Напишите **чисто виртуальный метод** получения символа клетки: `virtual char getSymbol() const = 0;` (Окончание `= 0` говорит компилятору, что у метода нет реализации. Это делает класс *абстрактным* — вы больше не сможете написать `Cell c;`, только `Cell* c;`).
2. Напишите метод взаимодействия: `virtual void onStep(Player& p) = 0;` (Что произойдет, если игрок наступит сюда?).
3. Напишите метод-разрешение: `virtual bool canWalk() const { return true; }` (По умолчанию по клетке ходить можно).
4. **Золотое правило C++:** `virtual ~Cell() {}` — Виртуальный деструктор обязателен, если класс предназначен для наследования! Без него при удалении массива будет утечка памяти.

### Шаг 2: Наследники (Оживляем мир)

Теперь создадим конкретные типы клеток. Они должны наследоваться от `Cell` и переопределять (слово `override`) его методы.

**1. Обычный пол (`Floor`):**
```cpp
class Floor : public Cell {
public:
    char getSymbol() const override { return '.'; }
    void onStep(Player& p) override { 
        // Ничего не происходит, просто стоим
    }
};
```

**2. Стена (`Wall`):**
Символ `'#'`, наступить нельзя (переопределите `canWalk()`, чтобы возвращал `false`), метод `onStep` оставьте пустым (ведь мы туда не наступим).

**3. Золото (`Gold`):**
Символ `'$'`. В методе `onStep` увеличивайте `p.gold += 10;`. 
*Проблема:* золото собрано, но оно останется на карте! Как сделать так, чтобы оно исчезло? 
*Решение:* добавьте полю `bool isCollected = false;` в класс `Gold`. Если собрано — возвращайте символ пола `'.'` и больше не давайте золота в `onStep`.

### Шаг 3: Карта (Массив указателей)

Создайте класс `Map`.
Внутри него будет массив **указателей** на базовый класс: `Cell* grid[10][10];`

1. **Конструктор:** Заполните карту. Сделайте двойной цикл `for`. По краям карты создавайте стены: `grid[y][x] = new Wall();`. Внутри — пол: `grid[y][x] = new Floor();`. А в пару случайных мест положите золото: `grid[5][5] = new Gold();`.
2. **Деструктор:** Мы использовали `new`! Значит, нужен двойной цикл `for`, где для каждой клетки вызывается `delete grid[y][x];`.
3. **Метод отрисовки `void draw(const Player& p) const`:** 
   Снова двойной цикл. Внутри проверяем: если текущие `x` и `y` совпадают с координатами игрока (`p.x` и `p.y`), выводим в консоль `'@'`. Иначе обращаемся к клетке через указатель: `cout << grid[y][x]->getSymbol();`. В конце каждой строки (внешнего цикла) делайте `cout << endl;`.

### Шаг 4: Игровой цикл и Логика движения

Добавьте в класс `Map` метод `void movePlayer(Player& p, int dx, int dy)`.

**Алгоритм движения:**
1. Вычисляем новые координаты: `int nx = p.x + dx;` и `int ny = p.y + dy;`.
2. Проверяем, не вышли ли мы за пределы массива (0..9).
3. **МАГИЯ ПОЛИМОРФИЗМА:** Проверяем, можно ли туда идти: `if (grid[ny][nx]->canWalk())`. Мы не знаем, Стена там, Пол или Золото! Мы просто спрашиваем клетку.
4. Если идти можно: 
   * Меняем координаты игрока: `p.x = nx; p.y = ny;`
   * Наступаем на клетку: `grid[ny][nx]->onStep(p);`. (Снова магия: если там золото, оно само добавится в `p.gold`).

### Шаг 5: Финальный Тест-драйв

Скопируйте этот `main` для запуска игры. Для движения используйте клавиши W, A, S, D и Enter.

```cpp
#include <iostream>

// ... (здесь ваши классы Player, Cell, Floor, Wall, Gold, Map) ...

int main() {
    Player player;
    Map gameMap;
    char input;

    while (player.hp > 0) {
        // Очистка консоли (для Windows: "cls", для Linux/Mac: "clear")
        system("clear"); // или system("cls");

        std::cout << "HP: " << player.hp << " | Gold: " << player.gold << "\n";
        gameMap.draw(player);
        std::cout << "Move (w/a/s/d) and press Enter: ";
        std::cin >> input;

        if (input == 'w') gameMap.movePlayer(player, 0, -1);
        if (input == 's') gameMap.movePlayer(player, 0, 1);
        if (input == 'a') gameMap.movePlayer(player, -1, 0);
        if (input == 'd') gameMap.movePlayer(player, 1, 0);
        if (input == 'q') break; // Выход
    }

    std::cout << "Game Over!\n";
    return 0;
}
```

### 🧠 Почему это круто и как это связано с Qt?
Поздравляю! Вы только что написали архитектуру графического фреймворка. 
В Qt всё устроено **абсолютно так же**:
* Класс `Cell` — это аналог `QWidget` в Qt.
* Метод `getSymbol()` — это аналог метода `paintEvent()` в Qt (который рисует кнопку или текст).
* Метод `onStep()` — это аналог `mousePressEvent()` (обработка клика).
* А ваш класс `Map` — это главное окно программы, которое хранит список виджетов и вызывает их методы при нажатии мыши или перерисовке экрана!
