# Лабораторная работа: «Магия операторов. Создаем ядро физического движка (Вектор 2D)»

### Цель работы:
Вы уже умеете создавать классы и вызывать их методы. Но писать `c = a.add(b)` вместо обычного `c = a + b` — это неудобно и некрасиво. В Python вы делали это с помощью «магических методов» (например, `__add__`). В C++ этот механизм называется **перегрузкой операторов**. 

В этой работе мы напишем класс двумерного вектора `Vector2D`. Этот класс научит вашу программу складывать координаты, умножать их на скорость и выводить в консоль так же легко, как обычные числа `int` или `float`.

*Важно: Здесь слово «вектор» означает математический объект (стрелку с направлением и длиной), а не динамический массив.*

### Шаг 2: Сложение и вычитание (Учимся передавать по константной ссылке)
Давайте научим C++ складывать два вектора. Формула проста: $x = x_1 + x_2$, $y = y_1 + y_2$.

Добавьте внутри класса следующий метод:
```cpp
Vector2D operator+(const Vector2D& other) const {
    return Vector2D(this->x + other.x, this->y + other.y);
}
```

**Разбор синтаксиса (ПОЧЕМУ ТАК СЛОЖНО?):**
*   `Vector2D` (в самом начале) — это тип, который мы возвращаем. Результат сложения двух векторов — это новый вектор.
*   `operator+` — это имя функции. Да, прямо со знаком плюса.
*   `const Vector2D& other` — мы передаем второй вектор **по ссылке** (`&`), чтобы не тратить память на создание его копии. А слово `const` гарантирует, что мы случайно не изменим этот второй вектор внутри функции.
*   `const` (в самом конце) — говорит компилятору: «Эта операция не меняет тот вектор, к которому применяется» (ведь при сложении `a + b` ни `a`, ни `b` не меняются, создается новый `c`).

**Задание к Шагу 2:**

### Шаг 5: Вывод в консоль (Магия `friend`)
Вызывать метод `.print()` — прошлый век. Мы хотим писать так: `cout << v1 << endl;`

Оператор `<<` нельзя перегрузить как обычный метод внутри класса `Vector2D`, потому что слева от `<<` стоит не вектор, а объект консоли `cout` (его тип `std::ostream`). 

Поэтому мы используем **дружественную (friend) функцию**. Она пишется *внутри* класса, но не принадлежит ему. Слово `friend` дает ей право читать приватные `x` и `y`.

Скопируйте этот код внутрь класса `Vector2D`:
```cpp
friend std::ostream& operator<<(std::ostream& os, const Vector2D& v) {
    os << "(" << v.x << ", " << v.y << ")";
    return os;
}
```
*(Не забудьте подключить `#include <iostream>` в начале файла).*

### Шаг 6: Финальная проверка (Тест-драйв)

Замените ваш `main` на этот код. Если вы всё сделали правильно, программа скомпилируется без ошибок и выдаст ожидаемый результат.

```cpp
int main() {
    Vector2D playerPos(10.0, 5.0);
    Vector2D velocity(2.5, -1.0);
    float time = 2.0;

    // Считаем новую позицию: Позиция + (Скорость * Время)
    Vector2D newPos = playerPos + (velocity * time);

    std::cout << "Start position: " << playerPos << std::endl;
    std::cout << "Velocity: " << velocity << std::endl;
    std::cout << "Position after 2 seconds: " << newPos << std::endl;

    Vector2D target(15.0, 3.0);
    if (newPos == target) {
        std::cout << "Target reached!" << std::endl;
    } else {
        std::cout << "Missed the target." << std::endl;
    }

    return 0;
}
```

**Ожидаемый вывод:**
```text
Start position: (10, 5)
Velocity: (2.5, -1)
Position after 2 seconds: (15, 3)
Target reached!
```
