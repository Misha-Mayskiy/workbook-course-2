### Шаг 1: Подготовка рюкзака (Выделение памяти)

В C++ есть два вида памяти: **Стек** (быстрая, но маленькая и автоматическая) и **Куча** (огромная, но ручная). Чтобы создать массив неизвестного заранее размера, нужно просить память в «куче» с помощью оператора `new`.

Создайте класс `Inventory`.
У него должно быть три приватных поля:
*   `int* data` — указатель на начало нашего массива в памяти.
*   `int size` — сколько предметов сейчас лежит в инвентаре (изначально 0).
*   `int capacity` — вместимость (на сколько элементов сейчас выделена память).

1. **Конструктор:** Напишите конструктор `Inventory(int cap = 5)`. Внутри установите `capacity = cap`, `size = 0`, и главное — выделите память: `data = new int[capacity];`.
2. **Деструктор:** Если мы взяли память через `new[]`, мы **обязаны** вернуть её системе при уничтожении объекта, иначе произойдет *утечка памяти*. Напишите деструктор `~Inventory()`, внутри которого напишите `delete[] data;`.
3. Напишите метод `void print() const`, который циклом выводит все элементы от `0` до `size - 1`.

### Шаг 2: Магия расширения (Метод Add)

Теперь самое интересное. Герой находит новый предмет, но инвентарь может быть полон.
Напишите метод `void add(int itemID)`. 

**Алгоритм внутри метода:**
1. Если место еще есть (`size < capacity`), просто кладем предмет: `data[size] = itemID; size++;`
2. **Если места нет (`size == capacity`):**
   * Создаем новый указатель на кусок памяти в 2 раза больше: `int* newData = new int[capacity * 2];`
   * Копируем (через цикл `for`) все старые предметы из `data` в `newData`.
   * **Важно:** Уничтожаем старый тесный массив! `delete[] data;`
   * Переключаем наш указатель на новый массив: `data = newData;`
   * Увеличиваем вместимость: `capacity *= 2;`
   * Теперь место есть! Кладем предмет: `data[size] = itemID; size++;`

*Проверьте в `main`: создайте инвентарь, добавьте 10 предметов (циклом) и выведите на экран. Вместимость по умолчанию была 5, значит массив должен был успешно расшириться сам!*

### Шаг 3: Катастрофа! (Ломаем программу)

Добавьте в `main` следующий код:
```cpp
Inventory hero1;
hero1.add(101); // Дали герою меч
hero1.add(102); // Дали щит

// Создаем второго героя и отдаем ему копию инвентаря первого
Inventory hero2 = hero1; 

hero2.print();
```

Запустите программу. Скорее всего, она **с аварийным треском упадет** (в консоли вы увидите что-то вроде *double free or corruption* или *segmentation fault*, или программа просто молча прервется).

**ПОЧЕМУ ЭТО ПРОИЗОШЛО? (Обязательно прочитайте!)**
В Python это сработало бы (создались бы две ссылки на один список). В C++ компилятор по умолчанию делает *поверхностное копирование*. Он скопировал значение указателя `data`. 
Теперь и `hero1`, и `hero2` указывают на **один и тот же кусок памяти**. 
Когда программа завершается, вызывается деструктор `hero2`, который делает `delete[] data`. Память очищена! А через секунду вызывается деструктор `hero1` и *пытается снова удалить ту же самую память*. Это фатальная ошибка операционной системы.

### Шаг 4: Конструктор копирования (Глубокая копия)

Чтобы починить ошибку, мы должны объяснить C++, как правильно копировать наш объект. Мы должны выделить *новую* память для второго героя и скопировать туда сами предметы, а не только адрес.

Добавьте в класс **Конструктор копирования**:
```cpp
Inventory(const Inventory& other) {
    this->size = other.size;
    this->capacity = other.capacity;
    
    // ВЫДЕЛЯЕМ НОВУЮ ПАМЯТЬ!
    this->data = new int[this->capacity];
    
    // Копируем предметы из other в this
    for (int i = 0; i < this->size; i++) {
        this->data[i] = other.data[i];
    }
}
```
*Запустите программу. Ошибка должна исчезнуть! Теперь у каждого героя свой независимый инвентарь.*

### Шаг 5: Оператор присваивания (Правило Трёх)

Мы починили создание копии при инициализации (`Inventory h2 = h1;`). Но что если объекты уже созданы, и мы делаем так: `h2 = h1;`? Упадет снова! Для этого нужна перегрузка оператора `=`.

**Правило Трёх в C++ гласит:** *Если вашему классу нужен ручной деструктор (для `delete`), то ему на 100% нужны и конструктор копирования, и оператор присваивания.*

Добавьте оператор присваивания:
```cpp
Inventory& operator=(const Inventory& other) {
    // 1. Защита от самоприсваивания (если написали h1 = h1;)
    if (this == &other) {
        return *this;
    }

    // 2. Очищаем нашу старую память (чтобы не было утечки)
    delete[] this->data;

    // 3. Дальше делаем то же самое, что в конструкторе копирования!
    this->size = other.size;
    this->capacity = other.capacity;
    this->data = new int[this->capacity];
    for (int i = 0; i < this->size; i++) {
        this->data[i] = other.data[i];
    }

    return *this;
}
```
