
# Пошаговая инструкция: как написать калькулятор физических величин на C++

## 1. Что мы вообще хотим получить

Наша цель — написать программу, которая умеет работать с физическими величинами так, чтобы:

- у каждой величины было **число** и **размерность**;    
- можно было выполнять арифметические действия;    
- при умножении и делении размерность вычислялась автоматически;    
- при сложении и вычитании программа запрещала складывать несовместимые величины;    
- результат можно было печатать в понятном виде;    
- позже можно было перейти к вводу формулы как строки.    

Примеры того, что программа должна уметь:

- по формулам `F = m * a`, `v = s / t`, `rho = m / V` правильно вычислять размерности;    
- не позволять делать бессмысленные действия вроде `масса + скорость`;    
- печатать ответ как `kg*m/s^2`, `m/s`, `kg/m^3` и т. п.

# 4. С чего начать технически

Для первой версии не нужно делать много файлов.  
Сделайте **один консольный проект** и один файл `main.cpp`.

Причина проста: если ученик одновременно борется и с логикой программы, и с заголовочными файлами, и с линковкой, он быстро запутается.

Только когда первая рабочая версия будет готова, имеет смысл делить проект на:

- `Dimension.h / Dimension.cpp`    
- `Unit.h / Unit.cpp    
- `Quantity.h / Quantity.cpp`    

Но сначала — **один файл**.

# 5. Шаг 1. Понять, что такое размерность

Сначала нужно договориться, как мы будем хранить размерность.

## Самый простой вариант

Берем три базовые величины:

- `M` — масса    
- `L` — длина    
- `T` — время    

Тогда:

- масса: `M^1 L^0 T^0`    
- длина: `M^0 L^1 T^0`    
- время: `M^0 L^0 T^1`    
- скорость: `M^0 L^1 T^-1`    
- ускорение: `M^0 L^1 T^-2`    
- сила: `M^1 L^1 T^-2`    

Это значит, что размерность можно хранить как **три целых числа**.

# 6. Шаг 2. Сделать структуру `Dimension`

Пока не используйте класс. Начните с `struct`. Это проще.

```cpp
#include <iostream>
#include <string>
using namespace std;

struct Dimension
{
    int m; // степень массы
    int l; // степень длины
    int t; // степень времени
};
```

## Что такое `struct`

Это способ объединить несколько связанных данных в один объект.

Например, вместо трех отдельных переменных:

```cpp
int m, l, t;
```

мы храним их как один объект:

```cpp
Dimension d;
```

# 8. Шаг 4. Написать функции для работы с размерностями

Пока вы знаете функции — этого достаточно.

## 8.1. Проверка равенства размерностей

```cpp
bool equalDimension(Dimension a, Dimension b)
{
    return a.m == b.m && a.l == b.l && a.t == b.t;
}
```

## 8.2. Умножение размерностей

При умножении физических величин степени складываются.

Например:

$$
(M^1L^0T^0) \cdot (M^0L^1T^{-2}) = M^1L^1T^{-2}  
$$

Код:

```cpp
Dimension multiplyDimension(Dimension a, Dimension b)
{
    Dimension result;
    result.m = a.m + b.m;
    result.l = a.l + b.l;
    result.t = a.t + b.t;
    return result;
}
```

## 8.3. Деление размерностей

При делении степени вычитаются.

```cpp
Dimension divideDimension(Dimension a, Dimension b)
{
    Dimension result;
    result.m = a.m - b.m;
    result.l = a.l - b.l;
    result.t = a.t - b.t;
    return result;
}
```

# 11. Шаг 6. Добавить единицу измерения

Теперь нужно хранить не только размерность, но и саму единицу.

Например:

- метр    
- килограмм    
- секунда    
- ньютон    

Для этого создадим еще одну структуру.

```cpp
struct Unit
{
    string name;      // короткое имя: "m", "kg", "s"
    double factor;    // коэффициент перевода в SI
    Dimension dim;    // размерность
};
```

---

# 12. Что означает `factor`

Очень важный момент.

Мы будем хранить все числа **в единицах СИ**.

Примеры:

- `1 m` -> `1.0` в СИ    
- `1 cm` -> `0.01` в СИ    
- `1 min` -> `60.0` в СИ    

То есть `factor` — это множитель, который переводит значение в СИ.

Например:

```cpp
Unit meter = {"m", 1.0, {0, 1, 0}};
Unit centimeter = {"cm", 0.01, {0, 1, 0}};
Unit second = {"s", 1.0, {0, 0, 1}};
Unit minute = {"min", 60.0, {0, 0, 1}};
```

# 13. Шаг 7. Добавить физическую величину

Теперь нужна структура, которая будет хранить уже саму величину.

Например:

- `2 кг`    
- `5 м/с^2`    
- `10 Н`    

Простейший вариант:

```cpp
struct Quantity
{

    Dimension dim;    // размерность

```

Обратите внимание: мы пока **не храним имя единицы внутри Quantity**.  
Мы храним только:

- число в СИ,

Это хороший проектный выбор, потому что так все арифметические действия проще.

# 14. Шаг 8. Написать функцию создания величины из числа и единицы

Пусть пользователь хочет создать `200 cm`.  
В СИ это `2.0 m`.

Значит нужна функция:

```cpp
Quantity makeQuantity(double value, Unit unit)
{
    Quantity q;
    q.valueSI = value * unit.factor;
    q.dim = unit.dim;
    return q;
}
```

Проверка:

```cpp
int main()
{
    Unit meter = {"m", 1.0, {0, 1, 0}};
    Unit centimeter = {"cm", 0.01, {0, 1, 0}};

    Quantity a = makeQuantity(2.0, meter);
    Quantity b = makeQuantity(200.0, centimeter);

    cout << a.valueSI << endl; // 2
    cout << b.valueSI << endl; // 2

    return 0;
}
```

# 15. Шаг 9. Реализовать сложение и вычитание величин

Складывать можно только величины одной размерности.

Например:

- `2 м + 30 см` — можно;    
- `2 кг + 3 м` — нельзя.    

## Функция сложения

```cpp
Quantity addQuantity(Quantity a, Quantity b)
{
    if (!equalDimension(a.dim, b.dim))
    {
        cout << "Ошибка: нельзя складывать величины разных размерностей!" << endl;
        exit(1);
    }

    Quantity result;
    result.valueSI = a.valueSI + b.valueSI;
    result.dim = a.dim;
    return result;
}
```

## Функция вычитания

```cpp
Quantity subtractQuantity(Quantity a, Quantity b)
{
    if (!equalDimension(a.dim, b.dim))
    {
        cout << "Ошибка: нельзя вычитать величины разных размерностей!" << endl;
        exit(1);
    }

    Quantity result;
    result.valueSI = a.valueSI - b.valueSI;
    result.dim = a.dim;
    return result;
}
```

Пока `exit(1)` допустим. Это грубый, но понятный способ остановить программу при ошибке. Позже заменим его на исключения.

# 16. Шаг 10. Реализовать умножение и деление величин

Здесь проверка на совместимость не нужна. Размерность просто пересчитывается.

## Умножение

```cpp
Quantity multiplyQuantity(Quantity a, Quantity b)
{
    Quantity result;
    result.valueSI = a.valueSI * b.valueSI;
    result.dim = multiplyDimension(a.dim, b.dim);
    return result;
}
```

## Деление

```cpp
Quantity divideQuantity(Quantity a, Quantity b)
{
    if (b.valueSI == 0)
    {
        cout << "Ошибка: деление на ноль!" << endl;
        exit(1);
    }

    Quantity result;
    result.valueSI = a.valueSI / b.valueSI;
    result.dim = divideDimension(a.dim, b.dim);
    return result;
}
```

```cpp
int main()
```

# 18. Шаг 12. Научиться выводить результат в выбранной единице

Пока программа печатает только число в СИ и размерность в виде `M^... L^... T^...`.

Теперь нужно уметь выводить в нужной единице.

Например:
- длину в метрах или сантиметрах;    
- время в секундах или минутах.    

Нужна функция:

```cpp
void printQuantityInUnit(Quantity q, Unit unit)
{
    if (!equalDimension(q.dim, unit.dim))
    {
        cout << "Ошибка: величину нельзя вывести в этой единице!" << endl;
        return;
    }

    double value = q.valueSI / unit.factor;
    cout << value << " " << unit.name;
}
```

Проверка:

```cpp
Quantity x = makeQuantity(250.0, Unit{"cm", 0.01, {0,1,0}});
printQuantityInUnit(x, Unit{"m", 1.0, {0,1,0}});   // 2.5 m
```

# 22. Шаг 13. Превратить `Dimension` в класс

До этого было:

```cpp
struct Dimension
{
    int m;
    int l;
    int t;
};
```

Теперь можно сделать так:

```cpp
class Dimension
{
private:
    int m;
    int l;
    int t;

public:
    Dimension(int mValue, int lValue, int tValue)
    {
        m = mValue;
        l = lValue;
        t = tValue;
    }

    int getM() const { return m; }
    int getL() const { return l; }
    int getT() const { return t; }

    bool equals(const Dimension& other) const
    {
        return m == other.m && l == other.l && t == other.t;
    }

    Dimension multiply(const Dimension& other) const
    {
        return Dimension(m + other.m, l + other.l, t + other.t);
    }

    Dimension divide(const Dimension& other) const
    {
        return Dimension(m - other.m, l - other.l, t - other.t);
    }

    void print() const
    {
        cout << "M^" << m << " L^" << l << " T^" << t;
    }
};
```

# 23. Объяснение новых элементов

## `private`

Поля класса скрыты от внешнего кода.

Это значит, что никто не может случайно испортить объект так:

```cpp
d.m = 1000;
```

## `public`

Это те методы, которыми разрешено пользоваться снаружи.

## Конструктор

Это функция, которая вызывается при создании объекта.

```cpp
Dimension d(1, 0, 0);
```

## `const`

Если метод не меняет объект, хорошо писать `const`.

Пока можно запомнить простое правило:

**если метод только читает данные, а не меняет — ставим `const`.**

# 26. Шаг 16. Добавить перегрузку операторов

Это тот этап, где калькулятор становится красивым и «математическим».

Вместо:


## Что такое перегрузка операторов

Это возможность научить свои классы понимать стандартные операции:

- `+`    
- `-`    
- `*`    
- `/`

# 27. Как перегрузить `+` для `Quantity`

Пример:

```cpp
class Quantity
{
private:
    double valueSI;
    Dimension dim;

public:
    Quantity(double valueSIValue, Dimension dimValue)
        : valueSI(valueSIValue), dim(dimValue)
    {
    }

    double getValueSI() const { return valueSI; }
    Dimension getDimension() const { return dim; }

    Quantity operator+(const Quantity& other) const
    {
        if (!dim.equals(other.dim))
        {
            cout << "Ошибка: нельзя складывать несовместимые величины!" << endl;
            exit(1);
        }

        return Quantity(valueSI + other.valueSI, dim);
    }

    Quantity operator-(const Quantity& other) const
    {
        if (!dim.equals(other.dim))
        {
            cout << "Ошибка: нельзя вычитать несовместимые величины!" << endl;
            exit(1);
        }

        return Quantity(valueSI - other.valueSI, dim);
    }

    Quantity operator*(const Quantity& other) const
    {
        return Quantity(valueSI * other.valueSI, dim.multiply(other.dim));
    }

    Quantity operator/(const Quantity& other) const
    {
        if (other.valueSI == 0)
        {
            cout << "Ошибка: деление на ноль!" << endl;
            exit(1);
        }

        return Quantity(valueSI / other.valueSI, dim.divide(other.dim));
    }
};
```

Теперь можно писать так:

```cpp
Quantity v = s / t;
Quantity F = m * a;
Quantity sum = x + y;
```

# 28. Почему это хороший пример перегрузки операторов

Перегрузка операторов бывает плохой, если она делает код неочевидным.

Здесь она хорошая, потому что:

- `m * a` действительно похоже на физическую формулу;    
- `s / t` действительно означает скорость;    
- `x + y` допустимо только для совместимых величин.    

То есть запись становится и красивой, и физически осмысленной.

# 30. Шаг 18. Добавить словарь единиц

Теперь программа должна уметь находить единицу по имени.

Например:

- `"m"`    
- `"cm"`    
- `"kg"`    
- `"s"`    

Для этого удобно использовать `map`.

```cpp
#include <map>
```

Создаем словарь:

```cpp
map<string, Unit> units;
```

Но здесь появляется проблема: `map` при работе через `[]` требует, чтобы объект умел создаваться без аргументов.

Чтобы не усложнять, можно использовать вставку через `insert`, либо добавить пустой конструктор.

Простой путь:

```cpp
class Unit
{
private:
    string name;
    double factor;
    Dimension dim;

public:
    Unit() : name(""), factor(1.0), dim(0,0,0) {}

    Unit(string nameValue, double factorValue, Dimension dimValue)
        : name(nameValue), factor(factorValue), dim(dimValue)
    {
    }

    string getName() const { return name; }
    double getFactor() const { return factor; }
    Dimension getDimension() const { return dim; }
};
```

Теперь можно писать:

```cpp
map<string, Unit> units;

units["m"] = Unit("m", 1.0, Dimension(0,1,0));
units["cm"] = Unit("cm", 0.01, Dimension(0,1,0));
units["kg"] = Unit("kg", 1.0, Dimension(1,0,0));
units["s"] = Unit("s", 1.0, Dimension(0,0,1));
```

# 31. Шаг 19. Сделать функцию поиска единицы по имени

```cpp
Unit getUnitByName(const map<string, Unit>& units, const string& name)

    if (it == units.end())
    {
```

# 33. Шаг 21. Перевести обработку ошибок с `exit(1)` на исключения

Это уже более «правильный» C++.

## Что такое исключение

Это способ сообщить об ошибке не через печать текста и аварийное завершение, а через специальный механизм языка.

Подключаем:

```cpp
#include <stdexcept>
```

Пример:

```cpp
if (!dim.equals(other.dim))
{
    throw runtime_error("Нельзя складывать величины разных размерностей");
}
```

А в `main()`:

```cpp
try
{
    // основной код
}
catch (const exception& e)
{
    cout << "Ошибка: " << e.what() << endl;
}
```

Для учеников это может быть первым знакомством с исключениями.

# 34. Шаг 22. Добавить новые размерности

До этого мы использовали только `M, L, T`.

Но для электричества нужны еще хотя бы:

- `I` — сила тока    

Тогда `Dimension` придется расширить до четырех координат.

Например:

```cpp
class Dimension
{
private:
    int m;
    int l;
    int t;
    int i;
    ...
};
```

Теперь можно описывать:

- ток `A`: `(0,0,0,1)`    
- заряд `q = I*t`: `(0,0,1,1)`    
- напряжение `U = A/q`    
- сопротивление `R = U/I`    

Это хороший следующий этап после механики.

# 38. Как устроен парсер в общих чертах

Парсер лучше делать в три шага.

## Шаг 1. Лексический анализатор

Разбивает строку на токены.

Например строка:

```txt
m * a / t
```

превращается в последовательность:

- `m`    
- `*`    
- `a`    
- `/`    
- `t`    

## Шаг 2. Синтаксический анализатор

Понимает структуру выражения и строит дерево.

Например:

- корень `/`    
- слева `m * a`    
- справа `t`    

## Шаг 3. Вычисление дерева

Каждый узел вычисляет `Quantity`.

# 39. Какой парсер реально по силам школьникам

Для сложных грамматик применяют генераторы парсеров.  Но для нас проще применить **рекурсивный спуск**.

Нужны функции:

- `parseExpression()` — для `+` и `-`    
- `parseTerm()` — для `*` и `/`    
- `parseFactor()` — для числа, имени или скобок    

Это очень классическая и понятная схема.

# 40. Как организовать AST

AST — это дерево выражения. (Абстрактное Синтаксическое Дерево)

Базовый класс:

```cpp
class Expr

public:
```

# 41. Важные моменты

Очень важно, понимать не только код, но и физическую идею.

## 1. Размерность и физический смысл — не одно и то же

Например, работа и энергия имеют одинаковую размерность, но разный смысл.

## 2. Совпадение размерностей не гарантирует правильность формулы

Если формула размерностно согласована, это хорошо, но она все равно может быть физически неверной.

## 3. Хранение в СИ — это архитектурное решение

Оно упрощает все вычисления и уменьшает количество ошибок.

## 4. Программа не «понимает физику целиком»

Она проверяет согласованность и выполняет заложенные правила.
